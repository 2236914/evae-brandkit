Section 05
Do's & Don'ts
The rules that protect the editorial luxury aesthetic. Every designer and developer should review this before building.
✓ Do
Use Cormorant Garamond for all serif moments
Use DM Sans for all UI, buttons, overlines
Keep all buttons at 0px radius — always
Use burgundy only in the 5 defined touchpoints
Use tawny for all active states and progress
Use charcoal for all primary button fills
Use 0.5px stone borders on all cards
Render user's name in italic tawny in emotional contexts
Use prices always in Cormorant Garamond serif
✗ Don't
Never round buttons — 0px radius everywhere, no exceptions
Never use mauve — it is removed from this project entirely
Never use burgundy in a 6th place
Never use shadows anywhere in the system
Never use gradients anywhere
Never use teal, green, blue or any other hue
Never use Montserrat, Inter, Roboto or system fonts
Never use two display-size headings on the same screen
Never use dark backgrounds except booking confirmation and onboarding splash
This is a luxury medical editorial iOS app.
It should feel like opening Vogue, not like opening a medical portal.
